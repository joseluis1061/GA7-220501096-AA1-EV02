Evidencia de desempeño: Estándares de codificación

En esta evidencia se define el estándar de codificación que se utilizará durante el desarrollo del software. Este informe técnico tiene como propósito establecer normas claras que faciliten la legibilidad, mantenimiento y escalabilidad del código, siguiendo los principios de la programación orientada a objetos.

Estándares de codificación según el lenguaje seleccionado

En este proyecto se ha seleccionado Java como lenguaje principal de desarrollo. A continuación, se presentan los principales estándares que se adoptarán:

Convención de llaves

Se usará el estilo K&R para apertura y cierre de llaves. Ejemplo: if (condicion) { ... }

Indentación

Se utilizarán 4 espacios por nivel de indentación para mantener la claridad del bloque de código.

Espaciado

Después de comas, operadores y palabras clave como if, for, while, se añadirá un espacio para mejorar la legibilidad.

Longitud de línea

Se recomienda no superar las 100 columnas por línea. Si una línea es muy larga, se parte en varias líneas claras y lógicas.

Nomenclatura de variables, constantes, clases y métodos

Variables

Se usará la convención camelCase. Ejemplo: numeroUsuario

Constantes

Se escriben en mayúsculas con guiones bajos. Ejemplo: MAX_LONGITUD

Clases

Se usará PascalCase y debe ser un sustantivo representativo. Ejemplo: UsuarioController

Métodos

También en camelCase y deben comenzar con un verbo. Ejemplo: calcularPromedio()

Comentarios y buenas prácticas

Los comentarios deben ser breves, claros y colocarse solo cuando sea estrictamente necesario. Se recomienda:

  • ✔️ Comentarios en línea para aclarar lógica compleja
  • ✔️ Comentarios de bloque para describir el propósito de clases o métodos
  • ✔️ Uso de etiquetas TODO o FIXME para tareas pendientes

Ejemplo:

// Calcula el promedio de las notas ingresadas
public double calcularPromedio() { ... }
